# AI 翻译 App 工作文档

## 项目目标

构建一个 Android 大模型翻译 App。第一版聚焦文本翻译，支持云端 OpenAI 兼容接口，并预留本地 Hy-MT 小模型离线翻译能力。

## 当前范围

- 从空目录创建 Android 原生工程。
- 使用 Kotlin、Jetpack Compose、MVVM 思路实现第一版。
- 云端翻译采用 OpenAI Chat Completions 兼容协议。
- 离线模型采用首次下载到 App 私有目录的方案。
- UI 开发前必须先使用 imagegen 生成设计图。
- 悬浮窗和划词翻译仅写入后续规划，不进入第一版代码实现。

## 架构决策

- 翻译能力通过统一接口抽象，降低云端与离线实现的耦合。
- 离线推理当前采用 Llamatik/llama.cpp Android 封装，已用标准 Q4_K_M GGUF 完成真机验证。
- 配置持久化使用 DataStore。
- 历史记录使用 Room。
- 网络请求使用 OkHttp。
- UI 使用 Jetpack Compose Material3。

## UI 设计流程

1. 每次 UI 开发前，先更新 `task文档.md` 和 `todo日志.md`。
2. 使用 imagegen 生成高保真 UI 设计图。
3. 将设计图保存到 `docs/ui/`。
4. 再按设计图实现 Compose 页面。
5. 实现后对照设计图检查页面结构、状态、按钮、文本溢出和基础深浅色适配。

- R2 Bucket：`ai-translate-assets`。
- R2 正式下载域名：`https://download.204152.xyz`。
- 模型 Manifest：`/models/models.json`。
- 应用更新分发：Cloudflare R2，同一 bucket `ai-translate-assets`。
- 应用更新正式域名：`https://download.204152.xyz`。
- 应用更新 Manifest：`/releases/latest.json`。
- Debug 测试更新包路径格式：`/releases/ai-translate-{versionName}-debug.apk`。
- 当前 R2 发版脚本：`scripts/publish-r2-debug-update.ps1`，负责构建 Debug APK、计算 SHA256、生成 `docs/r2/latest.json`、上传 APK 和 manifest。
- 许可：正式发布前必须补充 Tencent HY License、Notice、免责声明和地域限制说明。

## 应用更新流程

- App 设置页从 `https://download.204152.xyz/releases/latest.json` 读取最新版本信息。
- 当 manifest 中 `versionCode` 大于本机 `BuildConfig.VERSION_CODE` 时，显示下载并安装入口。
- 更新包下载到 App cache 的 `updates/` 目录，完成后校验 `sizeBytes` 和 `sha256`。
- 校验通过后通过 FileProvider 暴露 APK，并拉起 Android 系统安装器。
- 如系统未授权“安装未知应用”，先跳转到该 App 的授权页，保留已下载 APK，用户返回后可继续安装。
- 后续 Debug 测试发版默认执行 `scripts/publish-r2-debug-update.ps1`；正式 release 发版时沿用同一套 R2 manifest 结构，仅替换构建产物和签名配置。

## 文本朗读实现记录

- 1.0.2 起，主翻译页、快速翻译卡片、历史详情和悬浮翻译窗统一走共享系统 TTS 控制器。

- 语言选择会先按用户选择或文本字符推断，再回退设备默认语言、英文、简体中文。
- 如果设备未安装或未启用系统文字转语音引擎，App 只提供修复入口，不内置第三方语音引擎。

## 离线推理验证结论

- 已尝试接入 Llamatik Android 依赖，APK 中已包含 `libllama.so`、`libllama_jni.so` 等 native 库。
- Hy-MT1.5-1.8B-2bit GGUF 文件完整，大小为 `600534880` 字节。
- GGUF 元数据包含 `general.architecture = hunyuan-dense`。
- 当前公开 Llamatik/llama.cpp 内核无法加载该 Hy-MT 2bit GGUF。
- AngelSlim 模型页已提供 Android Demo APK，但同时说明专用 `llama.cpp kernel` 仍在 coming soon。
