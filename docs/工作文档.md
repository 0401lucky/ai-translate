# AI 翻译 App 工作文档

## 项目目标

构建一个 Android 大模型翻译 App。第一版聚焦文本翻译，支持云端 OpenAI 兼容接口，并预留本地 Hy-MT 小模型离线翻译能力。

## 当前范围

- 从空目录创建 Android 原生工程。
- 使用 Kotlin、Jetpack Compose、MVVM 思路实现第一版。
- 云端翻译采用 OpenAI Chat Completions 兼容协议。
- 离线模型采用首次下载到 App 私有目录的方案。
- UI 开发前必须先使用 imagegen 生成设计图。
- 悬浮窗和划词翻译仅写入后续规划，不进入第一版代码实现。

## 架构决策

- 翻译能力通过统一接口抽象，降低云端与离线实现的耦合。
- 离线推理当前采用 Llamatik/llama.cpp Android 封装，已用标准 Q4_K_M GGUF 完成真机验证。
- 配置持久化使用 DataStore。
- 历史记录使用 Room。
- 网络请求使用 OkHttp。
- UI 使用 Jetpack Compose Material3。

## UI 设计流程

1. 每次 UI 开发前，先更新 `task文档.md` 和 `todo日志.md`。
2. 使用 imagegen 生成高保真 UI 设计图。
3. 将设计图保存到 `docs/ui/`。
4. 再按设计图实现 Compose 页面。
5. 实现后对照设计图检查页面结构、状态、按钮、文本溢出和基础深浅色适配。

## 模型资料

- 当前可用模型：HY-MT1.5-1.8B-Q4_K_M。
- GGUF 文件：`HY-MT1.5-1.8B-Q4_K_M.gguf`。
- 体积：约 1.13GB。
- 分发：首包不内置模型，用户首次启用离线翻译时从 Cloudflare R2 分片下载。
- R2 Bucket：`ai-translate-assets`。
- R2 正式下载域名：`https://download.204152.xyz`。
- 模型 Manifest：`/models/models.json`。
- 许可：正式发布前必须补充 Tencent HY License、Notice、免责声明和地域限制说明。

## 离线推理验证结论

- 已尝试接入 Llamatik Android 依赖，APK 中已包含 `libllama.so`、`libllama_jni.so` 等 native 库。
- Hy-MT1.5-1.8B-2bit GGUF 文件完整，大小为 `600534880` 字节。
- GGUF 元数据包含 `general.architecture = hunyuan-dense`。
- 当前公开 Llamatik/llama.cpp 内核无法加载该 Hy-MT 2bit GGUF。
- AngelSlim 模型页已提供 Android Demo APK，但同时说明专用 `llama.cpp kernel` 仍在 coming soon。
- 因此本 App 后续需要等待官方公开 Android 可集成内核/SDK/源码，或拿到 Demo 对应可复用推理层后再继续接入。
- 2026-05-10 已切换到腾讯官方标准 Q4_K_M GGUF，文件大小 `1133080512` 字节，SHA256 为 `4383AC0C3C8E476DE98FF979C2A3F069F8C4FB385E7860CF2D28DA896CC477C7`。
