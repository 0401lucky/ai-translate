# Agent 工作规范

## 语言

- 所有回复、文档、注释优先使用简体中文。
- 面向用户汇报时先说结论，少堆术语。

## 开发流程

1. 每次新增功能或优化前，先更新 `docs/task文档.md` 和 `docs/todo日志.md`。
2. UI 工作必须先使用 imagegen 生成设计图，并保存到 `docs/ui/`。
3. 设计图完成后，才能进入 Compose UI 实现。
4. 完成代码后优先运行构建、测试或可用的验证命令。
5. 验证结果必须写回 TODO 或 Task 文档。

## 工程原则

- 保持实现简单、清晰、可测试。
- 不保留无用旧代码。
- 对关键流程和复杂逻辑添加必要中文注释。
- 600MB 离线模型不内置进 APK，采用首次下载方案。

## 高风险操作

删除文件、批量移动、系统配置、全局依赖安装、生产 API 调用等操作前，需要获得明确确认。
